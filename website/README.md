# Nix Containers Website

A static site that displays every container image in this repo with its
description, README, source, vulnerability scan, and the helm charts that
consume it. Built entirely at build time by Nix; deployed to GitHub Pages.

## Architecture

- **Static HTML** generated by Nix at build time. No server, no database, no
  runtime data fetching beyond a single `images-data.json` for the index's
  client-side search.
- **Per-image pages** at `/images/<name>/index.html` (~3140 of them).
- **Index page** at `/index.html` with client-side search/filter (Tailwind v3
  + vanilla JS).
- **Vulnerability data** ingested from the latest successful
  `security-scan.yml` artifact at deploy time. Optional ŌĆö absent data
  degrades gracefully.
- **Chart cross-links** ("Used by") sourced from
  `../chart-image-mapping.nix`.
- **Theme**: dark, monospace accents. Tailwind v3 + JetBrains Mono webfont,
  both vendored (no CDN dependencies at runtime).

## Files

- `generate-site.nix` ŌĆö the Nix derivation. Reads images, mapping, and
  templates; runs Tailwind + render.py.
- `render.py` ŌĆö build-time templater. Substitutes `{{KEY}}` placeholders
  into `templates/*.html`; renders README via cmark and Nix source via
  pygments.
- `templates/index.html` ŌĆö index page template
- `templates/image-page.html` ŌĆö per-image page template
- `tailwind/{tailwind.config.js,input.css}` ŌĆö Tailwind v3 config + entry
- `static/app.js` ŌĆö client-side search/filter (loads `images-data.json`)
- `static/fonts/JetBrainsMono-*.woff2` ŌĆö vendored webfonts

## Quick start

```bash
# Build (from repo root)
nix build .#website
ls result/         # index.html, images/, images-data.json, static/

# Serve locally
python3 -m http.server -d result 8080
open http://localhost:8080
```

## Deploy

Pushed to `main` ŌåÆ `.github/workflows/deploy-website.yml` runs:
1. Downloads the latest successful `security-scan-results` artifact
2. Builds `.#website` with `--impure` so the derivation can read
   `SCAN_DATA_PATH` and `PAGES_BASE_PATH` via `builtins.getEnv`
3. Uploads `result/` to GitHub Pages

The Pages source must be set to **GitHub Actions** in repo Settings (not
"Deploy from a branch").

## Build inputs (env vars consumed by the derivation)

| Variable | Set by | Effect |
|----------|--------|--------|
| `SCAN_DATA_PATH` | `deploy-website.yml` | Directory containing `*-trivy.json` files. When present, per-image pages render a Vulnerabilities panel; index shows critical-CVE total. Optional. |
| `PAGES_BASE_PATH` | `deploy-website.yml` | URL base path (default `/`, set to `/<repo>/` in CI). Required because GH Pages project sites publish at `https://<org>.github.io/<repo>/`. |

Both require `--impure`. Local builds without env vars produce a fully-functional site rooted at `/`.

## Data extracted per image

Read from `images/<name>/default.nix` + `images/<name>/README.md`:

- `name` (directory)
- `description` from `org.opencontainers.image.description` label
- `version` from `org.opencontainers.image.version` (static or
  `pkgs.<x>.version` dynamic)
- `category` from `io.nix-containers.image.category` label
- `upstream` from `io.nix-containers.image.upstream` label
- `aliases` from `io.nix-containers.image.aliases` label
- `hasTest` (boolean ŌĆö does `test.nix` exist?)
- `readme` (full markdown, rendered to HTML at build time)
- `nixCode` (full default.nix, syntax-highlighted at build time)
- `usedByCharts` ŌĆö list of Helm chart names from
  `chart-image-mapping.nix`
- `scan` (when `SCAN_DATA_PATH` is set) ŌĆö severity counts from the latest
  Trivy JSON for this image
